<!-- section: mission -->
# 项目使命

本项目用于推进“基于深度学习的废弃物数据管理系统”毕业设计，目标是完成系统源码、毕业设计说明书、答辩材料和相关过程文档。系统应服务于垃圾分类智能化场景，提供图像识别、相似案例检索、分类知识查询、智能问答、图片理解、历史记录管理和知识测试能力。

<!-- section: audience_stakeholders -->
# 受众与相关方

主要相关方包括学生作者、指导教师、评阅教师和答辩委员会。系统最终用户假定为需要快速了解垃圾分类结果和分类依据的普通用户或垃圾分类管理场景使用者。

<!-- section: current_phase -->
# 当前阶段

当前处于项目资料接管与初始规划阶段。已读取并梳理开题报告和毕业设计任务书的主要内容，已初始化 RecallLoom 项目记忆，后续可进入源码骨架搭建、需求分析文档整理或论文初稿推进。

<!-- section: scope -->
# 范围

项目主线应围绕任务书中的七个模块展开：图像识别模块、相似搜索模块、文字检索模块、智能交流模块、图片理解模块、历史记录模块和知识测试模块。技术栈以 Flask、PyTorch、ResNet50、Qdrant、HTML/CSS/JavaScript 为主，并可使用 MySQL 或 SQLite 记录业务数据。

<!-- section: source_of_truth -->
# 事实来源

后续需求与验收优先依据《基于深度学习的废弃物数据管理系统 毕业设计任务书.docx》。开题报告提供研究背景、意义、资料调研、可行性分析和进度安排，但其中个别进度表述可能存在其他课题遗留内容，需要以任务书和实际系统目标校准。

当前已知核心指标包括：识别四大类垃圾，可回收物、有害垃圾、厨余垃圾、其他垃圾；测试集分类准确率不低于 90%；支持 PNG、JPG、JPEG，单文件不超过 16MB；图像识别响应时间目标 3 秒以内；相似检索返回相似度大于 65% 的参考图片，数据库规模 1000 条以上时检索时间目标 2 秒以内；知识测试题库至少 20 道。

<!-- section: core_workflow -->
# 核心工作流

进入项目后优先读取 .recallloom/rolling_summary.md、.recallloom/update_protocol.md，必要时读取本文件。涉及源码或正式文档修改时，应先对照任务书与现有文件，保持最小修改，使用简体中文说明，并在出现有意义的项目状态变化后更新 RecallLoom。

Python 相关操作必须使用项目本地 .venv。当前 .venv 主要用于运行 RecallLoom helper；后续如开始 Flask/PyTorch 开发，应继续在该虚拟环境中安装和运行依赖。

<!-- section: boundaries -->
# 边界与约束

不要把任务书没有要求的功能扩展成主线。优先完成七个核心模块和论文所需证据。涉及外部 API 时，不要把密钥写入仓库；应通过环境变量或本地配置管理。若后续需要训练深度学习模型或执行长时间计算，应按项目规则评估并准备合适的 Slurm 脚本。
