# 基于深度学习的废弃物数据管理系统毕业设计说明书（初稿）

## 摘要

随着城市生活垃圾产生量持续增长，传统垃圾分类依赖人工识别和经验判断，存在效率低、准确率不稳定、用户学习成本高等问题。本文设计并实现一套基于深度学习的废弃物数据管理系统，系统以 ResNet50 卷积神经网络为核心图像分类模型，结合 Qdrant 向量数据库、Flask 后端框架、HTML/CSS/JavaScript 前端界面，并调用 DeepSeek 与星火模型 API，实现图像识别、相似搜索、文字检索、智能交流、图片理解、历史记录和知识测试等功能。系统面向可回收物、有害垃圾、厨余垃圾和其他垃圾四类场景，能够辅助用户快速获得分类结果、置信度和分类依据，为垃圾分类智能化提供可行的软件实现方案。

关键词：垃圾分类；深度学习；ResNet50；Flask；Qdrant；向量检索

## 1 绪论

### 1.1 研究背景

我国城市化进程不断加快，生活垃圾产生量持续增长。垃圾分类是实现垃圾减量化、资源化和无害化的重要基础，但实际推广过程中仍存在居民分类知识掌握不均、人工分拣效率低、分类结果缺乏解释等问题。深度学习和计算机视觉能够从图像中自动提取高层语义特征，为垃圾分类提供智能化技术支撑。

### 1.2 研究意义

本课题将 ResNet50 图像分类、Qdrant 向量检索、大语言模型问答和多模态图片理解结合起来，既能提高垃圾图片分类效率，又能通过相似案例、分类依据和知识测试增强用户学习效果。系统具有一定工程实践价值，也能体现深度学习技术在环保信息化场景中的综合应用。

### 1.3 国内外研究现状

图像分类领域已从传统人工特征逐步发展到卷积神经网络方法。ResNet 通过残差连接缓解深层网络退化问题，在图像分类和特征提取任务中表现稳定。向量数据库如 Qdrant、Milvus 和 Faiss 能够支持高维特征快速检索。大语言模型和图像理解模型的发展，则为垃圾分类问答和复杂图片解析提供了新的交互方式。

## 2 需求分析

### 2.1 功能需求

系统按照任务书要求设计七个核心模块：

1. 图像识别模块：支持 PNG、JPG、JPEG 图片上传，单文件不超过 16MB，输出四类垃圾预测结果、置信度和分类依据。
2. 相似搜索模块：提取图片深度特征，基于 Qdrant 检索相似度大于 65% 的参考案例。
3. 文字检索模块：通过关键词查询垃圾分类信息、处理指南和典型示例。
4. 智能交流模块：调用 DeepSeek API，回答用户关于垃圾分类的问题。
5. 图片理解模块：调用星火模型 API，分析图片中的物品、材质、状态并给出分类建议。
6. 历史记录模块：保存识别时间、图片路径、识别结果和置信度，支持分页查看和删除。
7. 知识测试模块：内置不少于 20 道测试题，支持随机抽题、答题反馈和得分统计。

### 2.2 性能需求

系统目标是在测试集上达到不低于 90% 的分类准确率；图片识别响应时间目标为 3 秒以内；相似搜索在 1000 条以上向量数据规模下目标响应时间为 2 秒以内。外部 API 调用受网络和接口服务状态影响，应在失败时返回明确错误信息。

## 3 系统总体设计

系统采用前后端分离思想。前端使用原生 HTML、CSS 和 JavaScript 构建交互界面，通过 Fetch API 调用 Flask 后端接口。后端负责文件上传校验、模型推理、数据库访问、外部 API 调用和 JSON 响应封装。SQLite 用于保存历史记录，Qdrant 用于存储和检索图片特征向量。



### 3.1 系统功能结构

系统功能结构围绕任务书七个模块展开，前端负责交互入口，后端负责业务调度，模型服务、向量数据库和外部大模型 API 提供智能能力支撑。

| 模块 | 主要输入 | 主要输出 | 实现状态 |
| --- | --- | --- | --- |
| 图像识别 | PNG/JPG/JPEG 图片 | 预测类别、置信度、分类依据 | 接口已实现，待训练权重 |
| 相似搜索 | 查询图片 | 相似案例列表、相似度、参考类别 | 查询与建索引脚本已实现，待 Qdrant 数据 |
| 文字检索 | 关键词 | 分类信息、处理指南、示例 | 已实现内置知识库检索 |
| 智能交流 | 自然语言问题 | DeepSeek 回答 | 接口已实现，待真实密钥验证 |
| 图片理解 | 上传图片 | 星火图片分析结果 | 接口骨架已实现，待真实接口参数验证 |

### 3.2 后端接口设计

| 接口 | 方法 | 功能 | 关键约束 |
| --- | --- | --- | --- |

| `/api/similar-search` | POST | 查询相似图片案例 | 依赖模型权重和 Qdrant 服务 |
| `/api/search` | GET | 关键词检索垃圾分类知识 | 支持热门关键词提示 |

## 4 数据库与模型设计

### 4.1 数据库设计

历史记录表 `recognition_history` 保存图片路径、预测类别、置信度、四类概率、分类依据和创建时间。该设计能够满足用户回溯识别结果和系统测试记录分析的需要。

### 4.2 模型设计

图像识别模型采用 ImageNet 预训练 ResNet50。训练阶段冻结主干特征提取层，仅微调最后全连接分类层，将输出类别调整为可回收物、有害垃圾、厨余垃圾和其他垃圾四类。该策略能够降低训练成本，并适合毕业设计规模的数据集实验。

### 4.3 数据集整理

公开数据集可优先参考 TrashNet 和 TACO。由于公开数据集原始标签体系与国内四分类标准不完全一致，需要将样本清洗映射到 `recyclable`、`hazardous`、`kitchen`、`other` 四个目录。映射规则应在实验记录中说明，避免影响结果解释。



### 4.4 Qdrant 向量索引设计

系统使用 ResNet50 去除最后分类层后的深度特征作为图像向量表示。建索引脚本 `scripts/build_qdrant_index.py` 扫描四分类参考图片目录，将图片路径、文件名和类别作为 payload 写入 Qdrant。集合默认名称为 `waste_images`，相似度计算采用余弦相似度，接口层默认过滤相似度小于 0.65 的结果。


## 5 详细实现

### 5.1 图像识别实现

后端 `/api/recognize` 接口负责接收图片、校验格式和大小、保存文件、调用 ResNet50 分类服务，并将结果写入历史记录。若模型权重或类别映射不存在，接口返回“模型未就绪”错误，避免使用模拟结果替代真实识别。

### 5.2 相似搜索实现

相似搜索模块复用 ResNet50 去除分类层后的 2048 维深度特征，调用 Qdrant 检索相似图片。系统默认仅展示相似度大于 65% 的结果，以提高返回案例的参考价值。

### 5.3 智能交互实现

智能交流模块调用 DeepSeek Chat API，系统提示词限定其作为垃圾分类助手回答问题。图片理解模块调用星火视觉接口，将图片编码为 Base64 并提交分析请求。两类 API 密钥均通过 `.env` 配置，不写入源码。

### 5.4 前端实现

前端采用单页布局，分区展示图像识别、相似搜索、文字检索、智能交流、图片理解、历史记录和知识测试。页面使用响应式网格布局，保证桌面端和移动端内容不重叠、不遮挡。

## 6 系统测试

### 6.1 功能测试

测试内容包括非法文件上传、模型未就绪错误提示、文字检索、知识测试抽题与评分、历史记录分页和删除等。外部 API 在缺少密钥或调用失败时应返回可读错误，不导致服务崩溃。

### 6.2 模型测试

训练脚本将数据集划分为训练集、验证集和测试集，记录每轮损失、验证准确率和最终测试准确率。最终测试准确率用于验证是否达到任务书中“不低于 90%”的指标要求。

| 编号 | 测试内容 | 输入 | 预期结果 |

### 6.4 当前测试结果

当前已完成 Flask 后端基础测试，测试命令为 `.venv\Scripts\python.exe -m pytest -q`，结果为 10 个测试用例全部通过。测试已覆盖健康检查、知识题库数量、文字检索、模型未就绪提示、非法格式上传、超 16MB 上传限制、DeepSeek/星火缺配置错误、知识测试评分、历史记录分页删除和清空。由于公开数据集、训练权重、Qdrant 千级数据和真实 API 密钥尚未补齐，模型准确率、相似检索耗时和真实大模型调用结果仍需在后续实验阶段补充。

## 7 总结与展望

本文完成了基于深度学习的废弃物数据管理系统的设计与初步实现。系统综合应用图像分类、向量检索、智能问答和图片理解技术，能够为垃圾分类提供较完整的软件支持。后续可继续扩充训练数据、优化模型准确率、完善 Qdrant 数据管理工具，并增强多轮对话和个性化学习分析能力。
